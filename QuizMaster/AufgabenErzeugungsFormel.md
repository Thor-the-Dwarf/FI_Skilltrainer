# AufgabenErzeugungsFormel

## Kernformel

```md
Aufgabe = (Thema-Objekt × Perspektive) × Fragetyp × InteractionType × Format
```

## Bedeutung

Die Formel beschreibt, wie aus einem fachlichen Einzelgegenstand eine konkrete Aufgabe entsteht.

Die Lesereihenfolge ist:

1. Thema-Objekt  
   Worum geht es fachlich genau?
2. Perspektive  
   Unter welchem Blickwinkel wird dieses Thema-Objekt betrachtet?
3. Fragetyp  
   Welche Denkhandlung soll ausgeführt werden?
4. InteractionType  
   Wie wird die Aufgabe bearbeitet?
5. Format  
   Wie ist die Antwort- und Auswertungslogik aufgebaut?

Am Ende entsteht daraus eine konkrete prüfbare Aufgabe.

## Merksätze

```md
Thema-Objekt = Worum?
Perspektive = Wovon aus betrachtet?
Fragetyp = Welche Denkhandlung?
InteractionType = Wie wird bearbeitet?
Format = Wie wird geantwortet?
Aufgabe = konkrete Kombination daraus
```

## Wichtiger Hinweis

Die Formel ist ein Bauplan, kein blindes Voll-Kreuzprodukt.

Nicht jede Kombination ist sinnvoll.

Beispiele:

- `was_fehlt × luecke_fuellen` passt gut
- `ursache_folge_erkennen × reihenfolge_bestimmen` passt gut
- `mehrere_richtige_antworten_waehlen × single` passt schlecht
- `begriff_zu_definition × prioritaet_setzen` ist meist unpassend

Praktisch gilt also:

```md
Sinnvolle Aufgabe = nur zulässige Kombinationen aus der Formel
```

## Kurzbeispiele

### IPv4

- Thema-Objekt: Subnetzmaske
- Perspektive: Berechnung
- Fragetyp: `fehler_finden`
- InteractionType: `luecke_fuellen`
- Format: `single`

Ergebnis:

> In der Angabe `255.255.0.255` steckt ein Fehler. Korrigiere die falsche Subnetzmaske: `255.255.___.255`

### RAID

- Thema-Objekt: RAID 5
- Perspektive: Ausfallsicherheit
- Fragetyp: `aussage_bewerten`
- InteractionType: `kategorie_zuordnen`
- Format: `binary`

Ergebnis:

> Ordne die Aussage als richtig oder falsch ein: RAID 5 verkraftet den gleichzeitigen Ausfall von zwei Festplatten.

## Begriffe

- `Thema-Objekt`: die kleinste fachliche Einheit, zu der direkt eine Aufgabe gebaut werden kann
- `Perspektive`: der Blickwinkel auf dieses Thema-Objekt, zum Beispiel Funktion, Fehlerfall, Sicherheit, Vergleich oder Ablauf
- `Fragetyp`: die geistige Operation, zum Beispiel `aussage_bewerten`, `fehler_finden` oder `vergleich_treffen`
- `InteractionType`: die Bearbeitungsart, zum Beispiel `kategorie_zuordnen`, `reihenfolge_bestimmen`, `luecke_fuellen`, `prioritaet_setzen`
- `Format`: die Antwortlogik, zum Beispiel `single`, `multi`, `best`, `binary`
